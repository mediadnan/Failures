# Conclusion
Now that you have a basic idea of what ``failures`` offers, you can make use of it in your application to reduce 
the code needed to monitor application failures.

Note that this guide's examples are just dummy code that abstract the usage of each element, it was so for simplicity
reasons and to be as generic as possible to let you _the developer_ imagine how it can benefit you in your project.

Later, more concrete use-case examples can be added to this document to give motivation to users.
